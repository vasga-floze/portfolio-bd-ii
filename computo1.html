UNIDAD I

BASES DE DATOS NOSQL


Este fue mi cómputo favorito en esta materia, amé el poder explorar el mundo de las bases de datos no relacionales. Obviamente al inicio tuvimos una introducción generalizada para entrar en contexto, al principio costó comprender la idea o más bien desligar la idea estructurada que traía de las bases de datos relacionales. Pero cuando comprendí que pueden existir diferentes formatos como clave-valor, documentos, columnas y grafos, se abrió un mundo de nuevas posiblidades.

El mundo de las bases de datos NoSQL nos son de gran utilidad a la hora de trabajar con grandes volumenes de datos distribuidos.

La primera base de datos no relacional de la que estuvimos aprendiendo fue una clave-valor: Redis , en la que estudiabamos sobre una caracteristica peculiar y es que los datis no se guardan en el disco duro sino en la memoria principal, dato muy interesante. Una ventaja de redis es que permite escribir menos líneas de código para almacenar, obtener acceso y utilizar datos en las aplicaciones.

De Redis nos pasamos a MongoDB, una base de datos documental, en lugar de usar tablas y filas como en las bases de datos relacionales tradicionales, MongoDB hace uso de colecciones y documentos. Esta fue una de las bases de datos con las que más cómoda me sentí.

Posteriormente estuvimos estudiando y aprendiendo sobre Cassandra, la cual está diseñada para manejar big data. La característica principal de Cassandra es almacenar datos en múltiples nodos sin un solo punto de falla. Me encantó el concepto sobre el que trabaja Cassandra, saber que cualquier nodo puede estar inactivo en caso de fallo, pero que se pueden utilizar los datos almacenados en otro nodo, esto es super genial. Con respecto a esta base de datos me quede con ganas de seguir haciendo más pruebas, es super interesante.

Luego continuamos con Neo4j la cual es una base de datos de gráficos, es super genial, porque Neo4j puede descubrir patrones que serían difíciles de detectar con otros DB. Las consultas que se pueden realizar creando patrones o relaciones entre los grafos es algo que ayuda a cualquier empresa o inclusive industrias que buscan una solución a un crecimiento de datos exponencial.
